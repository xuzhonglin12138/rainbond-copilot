# Rainbond 故障排查指南

## 常见问题诊断流程

### 1. 组件启动失败

**症状**：组件状态显示 `abnormal`，无法正常访问

**诊断步骤**：
1. **查看组件状态**
   - 使用 `get-component-status` 工具
   - 检查状态字段和错误信息

2. **查看日志**
   - 使用 `get-component-logs` 工具
   - 重点关注最后 50-100 行日志
   - 查找 ERROR、FATAL、Exception 等关键词

3. **常见原因**：
   - **OOMKilled**: 内存不足，需要扩容
   - **CrashLoopBackOff**: 启动失败，检查配置和依赖
   - **ImagePullBackOff**: 镜像拉取失败，检查镜像地址和权限
   - **配置错误**: 环境变量、端口、挂载路径等配置错误

### 2. 内存溢出 (OOM)

**症状**：
- 日志中出现 "out of memory"、"heap limit" 等关键词
- 组件频繁重启
- 状态显示 OOMKilled

**解决方案**：
1. **确认内存使用情况**
   - 查看监控指标，确认内存使用率
   - 分析是否有内存泄漏

2. **扩容内存**
   - 使用 `scale-component-memory` 工具
   - 建议扩容 50%-100%（如 512MB → 1024MB）
   - **注意**：扩容需要重启组件，属于高风险操作，需要用户审批

3. **优化应用**
   - 检查代码是否有内存泄漏
   - 优化数据结构和算法
   - 增加垃圾回收频率

### 3. 服务无法访问

**症状**：
- 组件状态正常，但无法通过域名或 IP 访问
- 返回 502、503、504 等错误

**诊断步骤**：
1. **检查组件状态**
   - 确认组件是否真的在运行
   - 检查健康检查是否通过

2. **检查端口配置**
   - 确认端口是否正确开放
   - 检查端口协议（HTTP/TCP/UDP）
   - 验证容器内端口和对外端口映射

3. **检查网络策略**
   - 确认是否有网络策略限制
   - 检查防火墙规则
   - 验证服务发现是否正常

4. **检查网关配置**
   - 确认域名解析是否正确
   - 检查网关路由规则
   - 验证 SSL 证书是否有效

### 4. 性能问题

**症状**：
- 响应时间长
- CPU 使用率高
- 请求超时

**诊断步骤**：
1. **查看监控指标**
   - CPU 使用率
   - 内存使用率
   - 网络流量
   - 请求响应时间

2. **分析日志**
   - 查找慢查询
   - 检查错误率
   - 分析请求分布

3. **优化方案**：
   - **水平扩容**：增加实例数量
   - **垂直扩容**：增加 CPU 和内存
   - **代码优化**：优化算法和数据库查询
   - **缓存优化**：增加缓存层，减少数据库压力

### 5. 存储问题

**症状**：
- 磁盘空间不足
- 数据丢失
- 读写性能差

**诊断步骤**：
1. **检查存储使用情况**
   - 查看存储卷使用率
   - 确认是否达到配额上限

2. **检查存储配置**
   - 确认存储类型（本地存储/网络存储）
   - 检查挂载路径是否正确
   - 验证持久化配置

3. **解决方案**：
   - **扩容存储**：增加存储卷大小
   - **清理数据**：删除不必要的文件
   - **优化存储**：使用更快的存储类型
   - **数据备份**：定期备份重要数据

## 诊断工具使用

### get-component-status
**用途**：获取组件当前状态

**返回信息**：
- 组件名称
- 运行状态（running/stopped/abnormal）
- 内存配置
- CPU 配置
- 实例数量

### get-component-logs
**用途**：获取组件日志

**参数**：
- `name`: 组件名称
- `lines`: 日志行数（默认 50）

**使用场景**：
- 启动失败时查看错误信息
- 运行异常时分析原因
- 性能问题时查找慢查询

### restart-component
**用途**：重启组件

**风险等级**：高

**使用场景**：
- 配置更新后需要重启
- 组件卡死需要强制重启
- 内存泄漏需要释放资源

**注意事项**：
- 会导致服务短暂中断
- 需要用户审批
- 建议在低峰期操作

### scale-component-memory
**用途**：调整组件内存配置

**风险等级**：中

**使用场景**：
- OOM 错误需要扩容
- 内存使用率过高
- 性能优化

**注意事项**：
- 需要重启组件
- 需要用户审批
- 建议逐步扩容，避免资源浪费

## 最佳实践

### 预防性监控
- 设置合理的告警阈值
- 定期检查资源使用情况
- 关注错误日志和异常指标
- 建立故障响应流程

### 快速定位问题
1. 先看状态，再看日志
2. 从最近的变更入手
3. 对比正常和异常时的差异
4. 使用排除法缩小范围

### 安全操作
- 高风险操作必须审批
- 重要操作前先备份
- 在测试环境验证方案
- 准备回滚预案
